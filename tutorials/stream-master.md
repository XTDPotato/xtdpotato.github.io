## 什么是**串流**？

> 一句话大白话：串流就像给你的高性能电脑开了一条「专属低延迟远程通道」—— 把电脑的画面、声音实时压缩传输到你的手机/平板/低配设备上，同时把你在这些设备上的按键、触屏、手柄操作，以毫秒级速度回传给电脑执行，让你随时随地都能用上主力机的满血性能。

### 🎮 串流 ≠ 普通投屏，核心区别在这里
很多人会把串流和手机投屏搞混，二者本质完全不同，体验更是天差地别：
- **传输方向**：普通投屏是**单向画面传输**（只能把电脑画面投到设备上）；而游戏串流是**双向闭环传输**（画面+声音传到设备，操作指令实时回传到电脑）
- **操作逻辑**：普通投屏只能看画面，所有操作必须回到电脑前完成；而串流可以直接在设备端操作，和坐在电脑前用本地键鼠/手柄的体验完全一致
- **延迟表现**：普通投屏普遍有300ms以上的延迟，只能用来看视频、看文档；而Moonlight串流优化后延迟可低至10ms以内，玩动作游戏、FPS、音游都不会有拖影和卡顿
- **画质上限**：普通投屏大多只支持1080P 30帧，画质压缩严重；而串流支持4K 120Hz HDR全规格，画质和本地玩电脑几乎没有肉眼可见的差别
- **性能依赖**：普通投屏需要投屏设备也承担一定的渲染压力；而串流的所有渲染、计算工作全由主力电脑完成，控制端只需要负责解码播放，哪怕是十年前的低配设备也能流畅跑

### 🔄 串流的完整工作流程（形象化拆解）
串流的整个过程，是一个毫秒级的闭环，就像一场「实时接力赛」，每一步都在和延迟赛跑：
1. **主力电脑（服务端/被控端）渲染画面**：你的高性能电脑正常运行游戏/软件，显卡满血渲染出高画质、高帧率的游戏画面，同时采集系统声音
2. **硬件编码压缩**：显卡的专用编码单元（NVIDIA NVENC/AMD AMF/Intel QSV），把画面和声音实时压成极小的视频流（类似直播推流，但只给你一个人专属传输）
3. **网络传输**：通过局域网/公网/虚拟局域网，把压缩后的音视频流，稳定传输到你的手机/平板/低配电脑（客户端/控制端）
4. **硬件解码播放**：控制端收到视频流后，用自身的硬件解码器，实时还原出高清画面和声音，同步显示在屏幕上
5. **操作指令回传**：你在控制端按手柄、点触屏、敲键盘的所有操作，会被打包成极小的指令包，以最快速度回传给主力电脑
6. **闭环执行刷新**：主力电脑收到操作指令后，立刻在游戏/软件中执行，渲染出新的画面，再重复上面的流程，全程循环仅需十几毫秒

### 📊 关于串流的性能损耗，你需要知道的
很多人担心串流会影响电脑性能，这里给你讲清楚所有细节：
- **核心损耗来源**：串流的性能损耗，主要来自**显卡的硬件编码环节**——显卡需要一边渲染游戏画面，一边把画面编码成视频流，这个过程通常会占用**5%-10%的显卡性能**，具体取决于你的显卡型号和编码设置
- **关键划重点**：现代显卡（NVIDIA 10系以上、AMD RX5000系以上、Intel 7代酷睿以上）都有**独立的硬件编码单元**，编码工作完全由这个独立单元完成，几乎不会占用游戏渲染的核心性能，对游戏帧率的影响微乎其微
- **怎么把损耗降到最低？**
  1. 必须开启**硬件编码**，绝对不要用软件编码（软件编码会吃大量CPU性能，延迟和帧率都会崩）
  2. 分辨率和码率不用盲目拉满：手机/平板这类小屏设备，1080P 20Mbps的码率，画质已经和原画几乎没有区别
  3. 优先选择**HEVC(H.265)** 编码：同等画质下，比传统H.264编码占用的带宽更低，编码压力更小，损耗也更低

## 2. 服务端（被控端）搭建
1. 安装 **Sunshine** 服务端后，在任务栏后台找到图标，右键选择 **Open Apollo**，原版选择**Open Sunshine**。
2. 或直接打开 Web 配置界面：[https://localhost:47990](https://localhost:47990)
3. **首次登录必须设置用户名和密码，请务必牢记**，忘记只能重装。

![第一次进入](icons/moonlight_c1.png)

4. 首次打开可能提示虚拟手柄未安装，需下载安装：
   - GitHub 下载：[ViGEmBus](https://github.com/nefarius/ViGEmBus/releases)
   - 备用下载：[123盘](https://www.123684.com/s/RzbrVv-CX1Kd)

5. **设置中文**：
   点击顶部 **Configuration** → 在 **Locale** 中选择中文 → 点击底部 **Save** → 点击 **Apply** 刷新生效。

![设置语言](icons/moonlight_c2.png)

6. **推荐配置**：
   开启 **UPnP 自动端口转发** 和 **IPv6 支持**，使用公网 IPv6 串流必须开启 IPv6。

![打开IPv6和UPnP](icons/moonlight_c3.png)

## 3. 客户端（控制端）连接
安装 **Moonlight** 或 **月光阿西西**（推荐阿西西改版），操作基本一致。

### 连接方式
输入 **IP 地址**：
- 局域网：使用内网 IP
- 异地/公网：使用 **皎月连** 虚拟局域网 或 公网 IPv6

![阿西西改版添加电脑下输入IP地址](icons/moonlight_c4.png)

## 4. 密钥配对（PIN 码验证）
1. 客户端点击电脑，记录显示的 **4 位 PIN 码**。
2. 回到 Sunshine/Apollo Web 配置：[https://localhost:47990](https://localhost:47990)
3. 点击顶部导航栏 **PIN码**。
4. 输入 PIN 码与设备名称（名称为空则使用默认），发送完成配对。
5. 配对成功后滑动到底部授予权限（Apollo 专用），首次连接默认拥有全部权限。

> 注意：PIN 码 5 分钟内未输入会失效，后台挂起也会刷新，失效后重新操作即可。

![密钥配对流程](icons/moonlight_c5.png)

---

## 5. 配置完成后
配对成功后，客户端即可点击进入桌面。

### 解决无声问题
若控制端没有声音：
1. 被控端右键右下角声音图标，打开 **声音设置**。
2. 在输出设备中找到 **Steam Streaming Speakers**（虚拟声卡）。
3. 选择**禁用**，重启 Sunshine/Apollo 或重新连接串流即可。

![禁用虚拟声卡](icons/moonlight_c6.png)

### 恢复声音
若需要双端发声：
1. 打开 **控制面板 → 硬件和声音 → 声音**。
2. 右键 **Steam Streaming Speakers** → 选择**启用**。
3. 重启 Sunshine/Apollo 生效。

![启用虚拟声卡](icons/moonlight_c7.png)

## 6. 进阶操作（异地/公网串流）
如果你没有公网 IP，或不想在局域网内串流，可使用 **皎月连** 虚拟局域网实现异地串流。

### 皎月连使用步骤
1. 下载皎月连：[皎月连官网](https://www.natpierce.cn/pc/downloads/index_new.html)
2. **控制端与被控端都必须安装**。
3. Windows 端下载 Windows 版，安卓端下载安卓版。
4. 双端登录**同一账号**。
5. 被控端点击**服务端**，设置访问密码并启用服务。
6. 控制端点击**客户端**，连接对应电脑即可。
7. IP 地址默认使用：10.6.22.1

> 非同一账号可使用**识别码 + 连接密码**进行配对，操作相同。
